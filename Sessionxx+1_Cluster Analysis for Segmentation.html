Managing Customer Heterogeneity

Objectives of Cluster Analysis

  • Cluster analysis aims to classify customers into relatively homogeneous groups called segments (more generally referred to as clusters) based on the characteristics of the customers 

  • Customers within each segment (cluster) should be similar to one another, but different from objects in other segments 

  • Cluster analysis helps the researcher understand patterns of similarities and differences that exist among customers

Objectives of Cluster Analysis (cont.)

To be useful, a segmentation analysis should yield a manageable number of segments (between three and eight segments is frequently recommended). Remember that firms have limited resources (MP#4) and hence typically have to choose a few customer segments to serve.

Naïve Cluster Analysis

  • Clustering refers to a broad set of techniques for finding subgroups of observations within a data set

  • What you just did is called naïve cluster analysis

  • Naïve approaches typically do not use analytical techniques, instead relying on intuitive and easy-to-spot patterns that exist in data.

    • Pros: Easy to conduct and often requires little effort
    • Cons: Insights obtained are often subjective (rather than data-driven) which are potentially less accurate and meaningful

K-means Cluster Analysis

  • Marketers typically collect two types of data: 1) Bases variables; 2) descriptor variables

  • Bases variables stand for data related to consumer characteristics such as attitudes and behaviors

    • Bases variables are typically not readily available for firms -> Need to do a survey

  • Descriptor variables such as demographics (e.g., age, gender of consumers) are typically available from firms’ database

  • Bases variables are usually used for identifying and understanding segments

    • As such, K-means/hierarchical clustering requires bases variables

  • Descriptor variables are usually used for predicting target segments

K-means Cluster Analysis (cont.)

  • K-means clustering is the simplest and the most commonly used clustering method for splitting a data set into a set of k groups

    • k represents the number of groups pre-specified by the analyst

    • There isn’t a response variable -> an unsupervised method

    • It seeks to find relationships between the n observations without being trained by a response variable

  • We want observations in the same group to be similar and observations in different groups to be dissimilar

  • The basic idea behind k-means clustering consists of defining clusters so that the total intra-cluster variation (known as total within-cluster variation) is minimized

K-means Cluster Analysis (cont.)

  • The standard algorithm is to minimize the sum of squared Euclidean distances between items (i.e., customers) and the corresponding centroid:

    \[ W(c_{k})={\displaystyle \sum_{x_{i}\in C_{k}}(x_{i}-u_{k})^{2}} \]

    where:

  • \(x_i\) is a data point belonging to the cluster \(C_k\)

  • \(\mu_k\) is the mean value of the points assigned to cluster \(C_k\)

  • The algorithm iteratively solves the minimization problem

    • There are lots of advanced algorithms, but we will use the most basic one

Hierarchical Cluster Analysis

  • Hierarchical clustering is an alternative approach to k-means clustering for identifying groups in the dataset

  • It provides an intuitive tree-based representation of the observations, called a dendrogram

  • However, it requires much higher computational power and storage compared to K-means

    • Not suitable for big data

    • You will soon see that it will not work well even for a small data set

  • Like K-means, there are many algorithms for hierarchical clustering

  • The textbook describes hierarchical clustering as if it is used for a selection of the number of segments for K-means clustering; That’s not true

Hierarchical Cluster Analysis (cont.)

  • Agglomerative clustering (also known as AGNES (Agglomerative Nesting)) is the most common approach for hierarchical cluster analysis

  • It works in a bottom-up manner

    • Each object is initially considered as a single-element cluster (leaf)

    • At each step of the algorithm, the two clusters that are the most similar are combined into a new bigger cluster (nodes)

    • This procedure is iterated until all points are members of just one single big cluster (root)

Takeaways

  • A foundational assumption in marketing strategy is that all customers differ in their attitudes and behaviors

    • A successful marketing strategy must manage and exploit customer heterogeneity

  • The segmentation allows a firm to manage customer heterogeneity by segmenting potential customers into relatively homogeneous groups, based on individual preferences and needs.

  • The segmentation uses cluster analysis to help marketing managers understand

    • how to identify the number of customer segments in the market

    • how to profile those customer segments. 

  • We learned K-means cluster analysis using R as a tool for segmentation
